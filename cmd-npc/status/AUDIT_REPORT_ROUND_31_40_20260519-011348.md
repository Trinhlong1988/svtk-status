## Detailed evidence per round

### Round 31 — SQL schema 14 missing columns

**Bug:** Em ship 14 fields trong JSONL không có cột tương ứng trong SQL schema → DB INSERT sẽ fail hoặc dữ liệu mất khi ingest. Missing: `background, era_start_year, is_historical_figure, is_player, is_protagonist, is_questgiver, is_raid_extreme, mentor, pet_base_atk, pet_base_hp, pet_evolution_path, pet_loyalty_init, recolor_index, starting_class`.

**Fix:** Add 14 columns + appropriate types:
```sql
recolor_index       SMALLINT DEFAULT 0,
is_raid_extreme     BOOLEAN DEFAULT FALSE,
pet_base_hp         INTEGER,
pet_base_atk        INTEGER,
pet_loyalty_init    SMALLINT,
pet_evolution_path  JSONB,
mentor              VARCHAR(96),
background          TEXT,
starting_class      VARCHAR(32),
is_player           BOOLEAN DEFAULT FALSE,
is_protagonist      BOOLEAN DEFAULT FALSE,
is_historical_figure BOOLEAN DEFAULT FALSE,
era_start_year      INTEGER,
gender              VARCHAR(8) DEFAULT 'male',  -- promoted CHAR(1) → VARCHAR(8)
```

### Round 32 — skill_ids duplicates within NPC

**Bug:** 191 NPC có cùng skill_id 2+ lần trong `skill_ids` array. Vd: idx=452 trainer `[150, 110, 150, 103]` — skill 150 trùng.

**Cause:** `gen_skills` dùng `rng.randint(1, 165)` with replacement.

**Fix:** `rng.sample(range(1, SKILL_REGISTRY_COUNT + 1), n)` — sampling without replacement.

**Post-fix:** 0/10000 NPCs có duplicate skill_ids.

---

### Round 33 — Gender heuristic detection

**Bug:** `normalize_existing_entry` sets `e.setdefault("gender", "male")` → backfills ALL 438 existing NPCs as male. Em mislabel 48 female NPCs: Phùng Thị Chính, Bà Triệu, Dương Vân Nga, Trần Thị Dung, ...

**Fix:** `detect_gender_from_name(name)`:
- "Thị" as separate word → female
- Last word ∈ FEMALE_GIVEN_SET → female
- Starts with "Bà " → female
- Default → male

**Post-fix:** 46 existing female detected. Samples: Bà Chúa Liễu Hạnh, Mỵ Châu, Hai Bà Trưng, Phùng Thị Chính, Thiều Hoa.

### Round 34 — is_questgiver redundancy

**Bug:** Historical figure inject block sets BOTH `is_questgiver: True` AND `can_give_quest: True`. Both fields are same boolean → redundancy.

**Fix:** Drop `is_questgiver`. `can_give_quest` is canonical (used by other NPCs).

**Post-fix:** 0 NPCs have `is_questgiver` field.

### Round 35 — Historical figures background

**Bug:** 12 historical figures (Lê Lợi, Nguyễn Trãi, Lê Lai, Nguyễn Huệ, etc.) ship without `background` text. Only protagonist had one.

**Fix:** Each historical figure now has:
```python
"background": f"Nhân vật lịch sử {fig} thời {era_data['name']}. "
              f"Trị vì khoảng {era_data['start_year']}-{era_data['end_year']}."
```

**Sample:** "Nhân vật lịch sử Lê Lợi thời Lê Era (1428-1788). Trị vì khoảng 1428-1788."

### Round 36 — SQL missing CHECK constraints

**Bug:** SQL schema chưa có CHECK cho:
- `cultural_tag` — could insert invalid values
- `aggro_range` — could insert negative or huge
- `gender` — was CHAR(1) DEFAULT 'M' but data uses "male"/"female"
- `starting_class` — no enum constraint

**Fix:** Added 4 CHECK constraints + promoted `gender CHAR(1)` → `VARCHAR(8)` with `IN ('male', 'female', 'M', 'F')` backward compat.

### Round 39 — pet_evolution_path semantic

**Bug:** `pet_evolution_path = [idx, (idx+1000)%9999+1, (idx+2000)%9999+1]` produces random NPC indices that aren't necessarily evolved forms of the pet — semantically misleading. Brief example `[1234, 1235, 1236]` shows tier progression.

**Fix:** `pet_evolution_path = [idx]` only. Add `pet_evolution_path_note: "Multi-tier expansion deferred to CMD content design"`. Multi-tier evolution requires separate template entries; out-of-scope CMD NPC v1.1.

### Bonus — Cultural lock follow-up "Triệu Vân"

**Bug:** R32 fix introduced `rng.sample` but earlier generation also produced 2 NPCs named "Triệu Vân" (Triệu surname + Vân female_given). "Triệu Vân" matches both:
