## 🎯 BUG REAL phát hiện và fix

**R231 — JSON↔SQL naming mismatch** (Phương án 6+7 chung root cause)

| JSON key | SQL column (pre-fix) | SQL column (post-fix via view) |
|---|---|---|
| `_index` | `npc_index` | `npc_index AS "_index"` (in view) |
| `sceneId` | `scene_id` | `scene_id AS "sceneId"` (in view) |

**Fix applied:**
- Added `CREATE OR REPLACE VIEW npc_templates_json_view` aliasing columns to JSON keys
- Added JSON↔SQL field mapping documentation in SQL DDL
- 51 other fields match 1:1 (audit-only `_gender_inferred`/`_uuid_backfilled`/`_historical_flag_inferred` skipped)

**Downstream impact:**
- CMD_DB ingest scripts can `SELECT * FROM npc_templates_json_view` and INSERT JSON directly
- CMD_BOSS/QUEST/MAP can reference NPC by `"_index"` matching JSON
- Backward compat: snake_case `npc_templates` table still works for SQL-native consumers
