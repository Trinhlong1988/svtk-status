## Detailed evidence per round

### Round 71 — JSON sort_keys deterministic

**Bug:** `json.dumps(n, ensure_ascii=False)` không có `sort_keys=True`. Python 3.7+ preserves dict insertion order, so keys order depends on em insertion sequence in dict construction. Different code paths → different key order in JSONL → diff churn.

**Fix:** `json.dumps(n, ensure_ascii=False, sort_keys=True)` in `write_jsonl()`.

**Post-fix:** Sample NPC keys alphabetical ✓.

---

### Round 72 — UUID5 deterministic

**Bug:** Em used `uuid.uuid4()` for both existing UUID backfill and generated NPCs. `uuid4()` is random — every build produces different UUIDs → `uuid_canonical_sha256` differs → R68 replay reproducibility broken.

**Fix:** `stable_uuid_for(name, idx) = str(uuid.uuid5(DNS_NS, f"svtk-npc-{idx}-{name}"))`. Deterministic given (name, _index).

**Post-fix:** Sample NPC idx=1 "Trần Long (player default)" → `de9985f4-704b-...` (stable across runs).

### Round 75 — Name+era diversity tracking

**Bug:** Random name generation produces duplicates within same era. Pre-fix: 828 (name, era) duplicates (e.g., "Hồ Đức" 5× in f1 era, "Vũ Toàn" 5× in g1).

**Fix:** Track `name_era_tracker[era]` as set; pass to `vi_name(rng, gender, taken_set=...)`. Retry 12× (up from 8) until unique within era.

**Post-fix:** 11 dupes remaining (98.7% reduction). Reaching 0 needs even larger name pool.
