## Detailed evidence per round

### Round 91 — Protagonist dual flag INFO alert

**Bug:** Probe `Q5`: protagonist idx=1 has `is_protagonist=True` AND `is_player=True`. Semantically:
- `is_player` = controllable by client (multiplayer hook)
- `is_protagonist` = main story-canon hero

For SVTK v1.1 only Trần Long is both. v2 multiplayer may diverge (player-controllable mob, non-protagonist player character).

**Fix:** `alert_protagonist_dual_flag()` LOW INFO severity. Documents the dual flag for v2 design decision; no code change to flags (kept both for forward compat).

**Post-fix:** Alert file `LOW-20260519-105507-npc_protagonist_dual_flag.json` ships with 1 sample (idx=1 Trần Long).

### Round 92 — Validator sceneId gen-range check

**Bug:** Validator did not cover sceneId range for generated NPCs. Existing 59 sceneId orphan (out of 1..7817) is covered by separate `npc_existing_scene_id_orphan_R75` alert (R71.1 immutable). Gen-side previously untested.

**Fix:** Validator check #61 `sceneId_gen_range_R92`:
```python
gen_npcs = [n for n in npcs if n["_index"] > existing_count]
sid_out = [n for n in gen_npcs if not (1 <= n["sceneId"] <= MAP_SCENE_COUNT)]
checks.append(("sceneId_gen_range_R92", len(sid_out) == 0, {...}))
```

**Post-fix:** 0 gen violations. Check pass.

### Round 93 — Validator skill_ids range check

**Bug:** Validator did not cover skill_ids range. SKILL_165 registry defines 1..165 valid IDs; out-of-range would silently break downstream skill lookup.

**Fix:** Validator check #62 `skill_ids_range_R93` iterates all NPCs × skill_ids and asserts `1 <= sid <= 165`.

**Post-fix:** 0 violations. Check pass.

### Round 94 — Validator ai_behavior valid set

**Bug:** Validator did not assert ai_behavior is in closed enum. TYPE_BEHAVIOR table defines 8 valid values: idle/wander/aggressive/patrol/defensive/train/event_perform/gather. Out-of-set value would silently break AI dispatch.

**Fix:** Validator check #63 `ai_behavior_valid_R94` asserts `ai_behavior in VALID_BEHAVIORS_CHECK`.

**Post-fix:** 0 invalid. Check pass.

---

### Round 95 — Validator tier→level adherence

**Bug:** Validator did not check that each gen NPC's level falls within `NPC_TIER_RANGE[tier]`. Existing 195 tier-0-lv>10 anomaly is covered by R83 alert (R71.1 immutable). Gen-side previously had no formal validator.

**Fix:** Validator check #64 `tier_level_adherence_R95` iterates gen NPCs and asserts level ∈ NPC_TIER_RANGE[tier].

**Post-fix:** 0 gen violations (gen code already respects ranges via `rng.randint(lvl_min, lvl_max)` — check formalizes invariant).

---

### Round 96 — Validator registry split sum

**Bug:** No validator covered the invariant: `npc_main + npc_side + npc_lore + npc_generated == npc_full == TARGET_NPC (10000)`. Split desync (e.g., writing partial main jsonl) would silently corrupt downstream consumers.

**Fix:** Validator check #65 `registry_split_sum_R96` reads all 5 jsonl files, sums parts, compares.

**Post-fix:** main 208 + side 132 + lore 98 + generated 9562 = 10000 == full 10000 == TARGET_NPC. Check pass.

### Round 97 — Main era gender gap INFO alert

**Bug:** Probe R98 (first probe): historical figures per main era have male-skewed gender distribution:
- Lý: 5 male / 2 female (Ỷ Lan implied via "Sư Vạn Hạnh (huyền thoại)" pool — but classified male by heuristic)
- Trần: 7 male / 0 female
- Lê: 5 male / 1 female
- Tây Sơn: 5 male / 1 female (Bùi Thị Xuân)
- Nguyễn: 5 male / 0 female

`ERA_MAIN.key_figures` lists rely on standard history records (king-centric) and lack female figures.

**Fix:** `alert_main_era_gender_gap()` LOW INFO severity. Reports per-era breakdown + lists eras with zero female + suggests enrichment pool (An Tư Công Chúa for Trần, Ỷ Lan for Lý, Hồ Xuân Hương / Đoàn Thị Điểm for Nguyễn). Content team owns next-round decision.

**Post-fix:** Alert file `LOW-20260519-105507-npc_main_era_gender_gap.json` ships with per-era breakdown + 2 zero-female eras (Trần, Nguyễn).
