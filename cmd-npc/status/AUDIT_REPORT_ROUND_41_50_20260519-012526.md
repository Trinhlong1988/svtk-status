## Detailed evidence per round

### Round 41 — Existing historical NPCs unflagged

**Bug:** Source `NPC_438.jsonl` has 14+ named historical NPCs (Mai An Tiêm, Hai Bà Trưng, Bà Triệu (Triệu Thị Trinh), Đinh Bộ Lĩnh, Lê Hoàn, Lý Thái Tổ, Lý Thường Kiệt, Sư Vạn Hạnh, Trần Hưng Đạo, Trần Quốc Toản, Hùng Vương, Sơn Tinh, Thủy Tinh, Lang Liêu, ...) but NO `is_historical_figure` field. Downstream consumers can't filter historical content.

**Fix:** `EXISTING_HISTORICAL_NAME_HINTS` (50 known historical names). In `normalize_existing_entry()`, substring match → set `is_historical_figure=True` + `_historical_flag_inferred=True` + auto-generate background.

**Post-fix:** 57 NPCs flagged (45 existing + 12 generated). Samples: Hùng Vương 18, Sơn Tinh, Thủy Tinh, Mai An Tiêm, Lang Liêu.

---

### Round 42 — Systemic existing hp anomaly (366/438 = 83%)

**Bug:** Massive scale violation of R78 stat formula in existing source. Bonus probe found 366/438 (83%) NPCs có hp >3× or <0.3× R78 expected. Previous R27 only alerted protagonist; this is systemic.

**Fix:** `alert_existing_systemic_hp_anomaly()` writes HIGH alert with:
- anomaly_count, total_existing, anomaly_pct
- 10 sample anomalies with actual/expected/ratio
- Recommendation: source-side regenerate with R78 compliance OR document R78 as advisory

---

### Round 43 — Mentor type semantic mismatch

**Bug:** R83 protagonist mentor = "Sư Vạn Hạnh". Mentor relationship implies teaching/quest-giving role. Existing source idx=84 has `npc_type=townsmen` (semantic mismatch).

**Fix:** `alert_existing_mentor_type_mismatch()` writes MED alert flagging mismatch with expected types [quest_giver, lore_npc, trainer].

---

### Round 44 — Alert directory accumulation

**Bug:** Each build run writes 6-7 fresh alerts to `cmd-lead/alerts/` with build TS. After 4 builds = 24 stale alerts mixed with current. Validator/LEAD consumers see duplicates.

**Fix:** `sweep_old_npc_artifacts()` moves all `*npc*.json` files NOT containing current TS to `cmd-lead/alerts-processed/`. Pattern matches existing repo convention (alerts-escalated-resolved/, completions-resolved/ already exist).

**Post-fix:** 9 active alerts (current TS only), 33 archived.

---

### Round 45 — Completions accumulation

**Bug:** Same pattern as alerts — 5 PASS completion files from 5 prior builds.

**Fix:** Sweep `cmd-lead/completions/` → `cmd-lead/completions-resolved/` keeping only current TS.

**Post-fix:** 1 PASS file: `PASS-npc_extend_to_10000-20260519-012526.json`.

---

### Round 46 — Transaction log idempotency

**Bug:** `write_npc_template_transaction_log()` writes new entry every build even if NPC registry hash unchanged. Violates R74.B append-log semantics (should only log new actions).

**Fix:** Check latest tx log entry's `uuid_canonical_sha256`; if matches new computed hash → log "idempotent skip" + don't write duplicate.

**Note:** Em ran 3 actual builds with different data deltas (gender heuristic R33, historical flag R41, etc.) so 4 tx files reflect real changes. Future identical runs will skip.
