## 🎯 BUG REAL phát hiện (R251 — V58 control char)

**Phương án 12 property fuzz** generated NPC with `name = "name with\\ttab"` and `name = "name\\nwith newline"`.

**Verifier pre-fix:**
- V05_name_nonempty: `bool("name\\nwith newline".strip())` = True → PASS
- V06_name_nfc: NFC normalize preserves \\n → PASS
- V07_name_length: < 80 → PASS
- V41_no_cjk: no CJK → PASS
- V55_no_bom: no BOM → PASS
- V56_no_trim_ws: `"name\\nwith newline".strip()` = "name\\nwith newline" (no leading/trailing WS) → PASS

→ ALL CHECKS PASS but name contains illegal control char.

**FIX:** Added V58_name_no_control_char:
```python
chk(not any(ord(c) < 32 for c in nm), "V58_name_no_control_char")
```

Catches: `\\n`, `\\t`, `\\r`, NUL, all C0 control chars.

**Post-fix property fuzz:** 1000/1000 trials, 0 unexpected pass. Verifier saturated.
