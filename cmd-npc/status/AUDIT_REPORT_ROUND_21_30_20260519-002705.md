## Detailed evidence per round

### Round 21 — recolor_index vs palette_seed (brief naming mismatch)

**Bug:** Brief CMD_NPC_v1.1.md lines 224/257/263 spec:
```python
'sprite_template_id': ...,
'recolor_index': ...
```
Em ship field name `palette_seed` thay vì `recolor_index`. Downstream consumer (CMD SPRITE LoRA art) read brief → expect `recolor_index` → would find missing.

**Fix:** Add `recolor_index` alias trong cả normalize_existing_entry và gen_one_npc + injection blocks:
```python
"palette_seed": pseed,
"recolor_index": pseed,  # R21 brief compliance
```

**Post-fix:** 10000/10000 NPCs have both fields.

### Round 22 — Protagonist starting_class (brief line 219)

**Bug:** Brief specifies `'starting_class': 'novice'` cho protagonist. Em ship 0/10000 với field này.

**Fix:** `normalize_existing_entry()` augment idx=1:
```python
if e.get("_index") == 1:
    e["starting_class"] = "novice"
```

### Round 23 — Pet template fields (brief line 826-838)

**Bug:** 352 pettable NPCs thiếu pet template fields per brief:
```python
"pet_base_hp": 500,
"pet_base_atk": 80,
"pet_loyalty_init": 50,
"pet_evolution_path": [1234, 1235, 1236]
```

**Fix:** Add 4 fields to all pettable NPC dicts. Pet template `pet_base_hp/atk` = NPC stats; loyalty init = 50 default; evolution_path = deterministic 3-tier progression (this_idx, +1000, +2000 mod 9999).

**Post-fix:** 352/352 pettable NPCs có đầy đủ 4 fields.

### Round 24 — Transaction log directory (R74.B)

**Bug:** R74.B specifies "Transaction log mỗi action" cho tradeable entities. Em ship CMD_NPC nhưng KHÔNG có `cmd-npc/transaction_log/` directory hay bootstrap entry.

**Fix:** `write_npc_template_transaction_log()`:
- Create `cmd-npc/transaction_log/{TS}-template_ship-{uuid}.json`
- Record: action=`npc_template_ship`, actor=`cmd-npc-bot`, evidence=`template_count + uuid_canonical_sha256`
- Note: per-instance pet logs happen at runtime when bắt pet

**Post-fix:** 2 tx_log files (latest build + prior intermediate).

### Round 25 — Tier 9 distribution (R76 + NPC_TIER_TARGET)

**Bug:** NPC_TIER_TARGET says tier 9 = 2% of NPCs (~200 expected). Pre-fix: 18 tier 9 NPCs = 0.19% (91% deviation). Cause: `TYPE_TIER_ALLOWED` only allowed boss to reach tier 9 (boss weight 2% → tier 9 capped ≈ 16-20).

**Fix:** Expand TYPE_TIER_ALLOWED:
```python
"monster":     (1, 9),   # R25: raid extreme monsters allowed tier 9
"event_npc":   (0, 9),   # R25: raid event NPCs allowed tier 9
"boss":        (5, 9),   # unchanged
```

R76/R80 don't strictly forbid monster/event_npc reaching tier 9 (raid extreme content).

**Post-fix:** 56 tier 9 NPCs across 3 types (monster=39, boss=12, event_npc=5). Improvement 3.1× pre-fix. Still below 2% target but architecturally correct — full 2% would require boosting monster/event weights, which is content design choice (document in honest_gaps).

### Round 26 — era_extra_9.json empty declared eras

**Bug:** era_extra_9.json declared 9 extra eras (pre_lich_su, bac_thuoc_g1, ..., hoa_lu_968_origin) but ALL 9 = 0 NPCs (only f1-f5 + g1 populated).

**Fix:** Split structure:
```json
{
  "eras_declared": [...],
  "active": {f1, f2, f3, f4, f5, g1},
  "unused": {pre_lich_su: {"status": "DECLARED_NOT_POPULATED", "reason": "..."}}
}
```

Avoids silent empty buckets confusing downstream consumers.

### Round 27 — Protagonist hp anomaly alert

**Bug:** Protagonist idx=1 hp=100 với lv=10 tier=0 townsmen. R78 formula expects hp=(50+200)*1.0*0.2=50. Actual 2× formula → source anomaly.

**Fix:** `protagonist_check_alert()` raises LOW alert if hp ≥ 1.5× expected or ≤ 0.5× expected. R71.1 source immutable.

**Post-fix:** Alert `cmd-lead/alerts/LOW-{TS}-npc_protagonist_stats_anomaly.json` created with full evidence.

### Round 28 — aggro_range tier scaling (R80)

**Bug:** All combat NPCs have aggro_range=6 hardcoded. R80 boss tier 6-7 + thanh tier 7-8 + thần tier 9 should have larger aggro than tier 0 regular.

**Fix:** `TIER_AGGRO_RANGE = {0:4, 1:4, 2:5, 3:5, 4:6, 5:7, 6:8, 7:10, 8:12, 9:16}` indexed by tier.

**Post-fix:** Aggro values now 8 unique [4,5,6,7,8,10,12,16] across combat NPCs.

### Round 29 — is_raid_extreme marker
