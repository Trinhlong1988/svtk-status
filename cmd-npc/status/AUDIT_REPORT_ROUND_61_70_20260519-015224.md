## Detailed evidence per round

### Round 61 — Boss epithet 100% coverage

**Bug:** `if npc_type == "boss" and rng.random() < 0.7:` only added epithet 70% of time. 30% of bosses had bare given-name like "Nguyễn Hùng" — semantically weaker than "Nguyễn Hùng Đại Vương".

**Fix:** Remove `rng.random() < 0.7` gate. Always assign epithet for boss class.

**Post-fix:** 169/169 generated bosses have epithet (100%).

---

### Round 62 — Epithet-class semantic mapping

**Bug:** Original epithet pool was 8 names sampled uniformly. Result: "Yêu Vương" on thanh class (should be boss), "Hắc Đế" on boss (should be than), "Thần Tướng" on boss (should be thanh/than). 80 mismatches.

**Fix:** `EPITHET_BY_CLASS` mapping:
```python
"mini_boss": ["Tiểu Vương", "Bạch Hổ Tướng", "Yêu Tử"],
"boss":      ["Đại Vương", "Quái Tổ", "Yêu Vương", "Bạch Hổ Tướng"],
"thanh":     ["Thánh Tướng", "Thần Tướng", "Ma Tôn"],
"than":      ["Hắc Đế", "Thiên Thần", "Đại Thần", "Vạn Cổ Tôn"],
```

Compute class first, then pick from class-appropriate pool.

**Post-fix:** 0 mismatches.
